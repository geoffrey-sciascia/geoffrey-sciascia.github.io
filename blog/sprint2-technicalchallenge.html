What are the differences between relative, absolute, and fixed positioning?

A CSS position is a rule set for an element that ultimately controls how it is placed in the viewport. There are 5 CSS positions. They are; Static, Relative, Absolute, Fixed, and Sticky. Keywords used are; top, right, bottom, left. They are always cited in that order. They can have positive and negative values. What these values do, depend on the positioning style used.

Before I discuss the different between these 3 positions, I will explain the first or default position of...

Static

When you create an element in your html source code, but don't designate a position, the position of the element is set to 'Static'. When viewing this in a web browser, it places the element on the page where it has been placed in the workflow. This element itself is an example of Static positioning. Its has no CSS positioning rules applied to it and so has no movement from its' original position in the workflow.

Relative

A Relative position is the position of an element relative to its’ own placment in the workflow. Elements are NOT removed from document workflow. Other elements will not fill the empty space left in its’ original position. You would use this positioning style when you would like an element to appear statically in a position that is different from its' own original position in the document workflow. Sticky

This is newer positioning tool. As you scroll down the page, the element will move up the page until a predetermined position is reached on the screen, then it becomes static in that position on the page. All other content will scroll up beneath it. The element is treated as relatively positioned until its' containing block crosses a specific threshold (such as setting top value other than auto) within its’ flow root (or container it scrolls within), at which point it is treated as “stuck” until meeting the opposite edge of its’ containing block. -behaves like relative until they are “stuck”, then they are fixed.

Z-index (stacking order): -Determines what element will be scrolling beneath which element if there is more then 1 element. Set rules for each element class (Z-index: 1;). Higher ‘z-index’ numbers are on top of lower indexes.
