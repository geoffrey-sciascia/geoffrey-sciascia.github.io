What are the differences between relative, absolute, and fixed positioning?

w

A CSS position is a rule set for an element that ultimately controls how it is placed in the viewport. There are 5 CSS positions. They are; Static, Relative, Absolute, Fixed, and Sticky. Keywords used are; top, right, bottom, left. They are always cited in that order. They can have positive and negative values. What these values do, depend on the positioning style used.

Before I discuss the different between these 3 positions, I will explain the first or default position of...

Static

When you create an element in your html source code, but don't designate a position, the position of the element is set to 'Static'. When viewing this in a web browser, it places the element on the page where it has been placed in the workflow. This element itself is an example of Static positioning. Its has no CSS positioning rules applied to it and so has no movement from its' original position in the workflow. Imagine there was a group of people told to gather in a room and form a line in the order they came in. Each person would be positioned statically in the room.

Relative

A Relative position is the position of an element relative to its’ own placment in the workflow. Elements are NOT removed from document workflow. Other elements will not fill the empty space left in its’ original position. You would use this positioning style when you would like an element to appear statically in a position that is different from its' own original position in the document workflow. An example of the code used on the CSS file is;

{position: relative;

top: 50px;

right: 50px;}

An example of what this looks like;

This element has position:relative;

In this example, I have moved the element 50px to the left from its' original position in the center of the screen. Now imagine the group of people from before standing in a line. A new person, the director (CSS), comes into the room. They see the line of people and direct one person to move 2 steps forward, and 2 steps to the left. Nobody else in the room moves. The person who has moved is now positioned relatively from their original position by 2 steps forward and 2 steps to the left.

Absolute

An Absolute position is the position of an element relative to a parent element. Elements ARE removed from document flow. Other elements will fill the empty space left in its’ original position. This positioning style is used when you have an element nested inside of a 'parent' element, and would like to control how it appears in relation to the 'parent' element. An example of the code used on the CSS file is;

{position: absolute;

top: 50px;

right: 50px;}

PLEASE NOTE that the element this code is affecting must be nested in another element whose position is set to relative.

An example of what this looks like;

This element has position: relative;
This element has position: absolute;

In this example I have placed the 'absolute' box, 80px from the Right side and 80px from the top side of the 'relative' box. Imagining the group of people again. There is a 2 meter squared black boarded box painted on the floor. It is one meter away from the northern wall, and two meters away from the western wall. The director tells the fourth person in line to go to the south-western corner and take 3 steps north and 3 steps east. everybody else in the line behind them moves forward a space to fill the missing spaces in the line. The person in the box is placed absolutely 3 stpes north and east from the south-western corner of the box.

Fixed

A fixed position is the position of an element in relation to the viewport. Elements ARE removed from the workflow. Other elements will fill the empty space left in its’ original position. This positioning style would be used if you would like an element to appear on the page statically in relation to the viewport, and not move when scrolling up or down the page. An example of the code used on the CSS file is;

{position: fixed;

top: 50px;

right: 50px;}

An example of what this looks like would stay on this page at all times no matter where on it you have scrolled. It looks like a image floating on top of all other content on the page and doesn't move from its' position in the screen whether you're scrolling up, down, sideways, or even resizing the window.

Sticky

This is newer positioning tool. As you scroll down the page, the element will move up the page until a predetermined position is reached on the screen, then it becomes static in that position on the page. All other content will scroll up beneath it. The element is treated as relatively positioned until its' containing block crosses a specific threshold (such as setting top value other than auto) within its’ flow root (or container it scrolls within), at which point it is treated as “stuck” until meeting the opposite edge of its’ containing block. The element behaves like a relative element until it is “stuck”, then it is fixed. An example of the code used on the CSS file is;

{position: sticky;

top: 0px;}

An example of what this looks like is the header for the page. No matter how far down you scroll on the page, the header remains at the top, and everything else scrolls beneath it.

Z-index (stacking order)

The 'z-index' determines what elements will be scrolling beneath or above which other elements, if there is more than 1 element. Higher ‘z-index’ numbers appear on top of lower indexes. You set rules for each element tag, class, or id inside the CSS file using the code 'z-index: N;'. The 'N' value can be any number.
